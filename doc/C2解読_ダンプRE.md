# C2(Cryptomeria)暗号 実装の解読（メモリダンプからのRE）

最終更新: 2026-06-21 ／ 対象: SD-MobileImpact の `SDVM2TSPacketDecParser.dll`（実行時展開）

> 方法は一般化、**この事例の具体値（アドレス・定数）は明記**。出典: Process Hackerフルダンプ
> `SD-MobileImpact.exe.dmp`(206MB, MDMP) を再生中(PRG011再生中)に取得し解析。

## 0. 状況
- ワンセグ`.sb1`の暗号は **C2(Cryptomeria)**。S-box は実行時メモリから抽出済み（`doc/_migration_kit/c2_sbox.bin`、
  先頭 `f7 3a cd 29 fc 11 17 fe 39 36 86 48 f3 e2 af aa`）。
- `SDVM2TSPacketDecParser.dll`は**パック保護**で静的解析不可だが、**再生中のメモリダンプには展開後コードが存在**。
  ダンプ内でS-boxは11コピー、各コピーの直前後が**C2コード**（S-boxをコードに埋め込み、`call $+0x100;pop`で
  自アドレス取得→ローカルへ`rep movsd`コピーする位置独立実装。だから絶対参照0件だった）。
- ダンプ解析で **PRG011(NHKニュース7)が再生中**と確定（その暗号化`.sb1`がダンプ内 VA 0xb5600c0 に存在）。
  → タイトル鍵はこの録画のもの。

## 1. C2 ラウンド関数 F （VA 0x36014950, ダンプから逐語RE）
4バイト `W` を、4バイトのラウンド鍵 `K` と S-box `S[256]` で変換（in-place）:
```
W[0]^=K[0]; W[1]^=K[1]; W[2]^=K[2]; W[3]^=K[3]      # 鍵挿入(バイト単位XOR)
W[0]=S[W[0]]; W[1]=S[W[1]]; W[2]=S[W[2]]; W[3]=S[W[3]]   # 全バイトS-box
W[0] ^= W[1]^W[2]^W[3]                               # 線形混合
W[0] = S[W[0]]                              １を         # W0再S-box
W[1] ^=  W[0]
W[2]  = (W[2] - W[0]) & 0xff
W[3]  = (W[3] + W[0]) & 0xff
```
※ 別ラウンド用 `0x3601484b` も同型(先頭一致)。特許US9014370B2の記述（t=data+key の加算・低位バイトのみS-box）
とは異なる＝実装はこのダンプ版が真。

## 2. 鍵スケジュール （VA 0x360138d7, RE中）
- 入力 56bit鍵を 32bit×2 (key0=低位, key1=高位) として、初期 A,B を生成:
  ```
  A = key0 ^ (key0<<23) ^ (key0>>23) ^ (key1>>9) ^ (key1<<9)   # shl/shr 0x17=23, 9
  B = key1 ^ (key1<<23) ^ (key1>>23) ^ (key0>>9) ^ (key0<<9)
  ```
- 先頭ラウンド鍵 = A をビッグエンディアン4バイトで出力。
- ループ i=1..15（計16ラウンド鍵）: `tmp = (A<<i)^(B>>(32-i)); B=(B<<i)^(A>>(32-i)); 出力[i]=tmp(BE4)`（RE要確認）。
  → 計16個の32bitラウンド鍵を生成。

## 3. ブロック復号ボディ （VA 0x36012cec〜, decrypt(key, buf, len)）
- 引数: key[ebp+8], buf[ebp+0xc], len[ebp+0x10]。`len/8` 個の8バイトブロックをループ。
- 起動時に鍵スケジュール(0x360138d7)でラウンド鍵をローカル[ebp-0x44]へ生成。S-boxはローカル[ebp-0x14c]へコピー。
- 各8バイトブロック = 前半A(bytes0-3)・後半B(bytes4-7)。**XOR Feistel**:
  ```
  tmp=B; F(B, rk_i, S); B = B ^ A; A = tmp     # 1ラウンド。rk毎・F関数(0x36014950/0x3601484b)交互
  ```
  をラウンド数ぶん（要確定。S-box埋込で展開コードが長い＝アンロール）。
- モード(ECB/CBC)・最終格納・バイト順(エンディアン)は要確定。

## 4. 残タスク（復号完成まで）
1. 鍵スケジュールのループとボディのラウンド数/順序/モードを確定（or **Unicornで実コードをエミュレート**して
   再実装バグを排除）。
2. **タイトル鍵(56bit)の特定**：ダンプ内に必ず存在。候補:
   - 復号関数が再生中に呼ばれていれば、スレッドスタックの引数[ebp+8]が鍵ポインタ→鍵本体。
   - or 永続鍵構造(ヒープ)。or 正しいC2での鍵総当たり(ダンプ内8byte窓×.sb1のTS化判定0x47)。
3. 鍵確定後、コピー済み`.sb1`をホストでオフライン復号→クリーンTS。録画ごとに鍵を取得。

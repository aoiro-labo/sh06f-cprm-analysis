# デジタル放送録画データの長期保存に関する政策提言

最終更新: 2026-06-23

> 本書は SH-06F ワンセグ録画の CPRM 解析プロジェクトにおける実体験から導き出した提言です。
> 技術的根拠は [`暗号構造解析.md`](暗号構造解析.md)・[`SD-MobileImpact解析.md`](SD-MobileImpact解析.md) を参照。

---

## 1. 現状の危機：ワンセグ/フルセグ録画が2026年に詰む理由

### 1-1. 何が起きているか

2025〜2026年、携帯電話（SH-06F 等 ISDB-T 対応 Android 端末）で録画した映像データが、
**正規手段では事実上再生・救出できない状態**になっている。

```
問題の連鎖:

SD-MobileImpact (正規CPRM再生ソフト)
  → 2018/3 配布終了 → 新規インストール不可
  → 動作 OS は Windows XP のみ (2014/4 EOL)
  → Vista 対応パッチのサーバも消滅
  → Windows 10/11 で OS レベルの CPRM サポート撤去

録画機端末 (SH-06F 等)
  → 製品終息、修理不可
  → 端末が壊れれば元 SD の再生手段がなくなる

結果: 正規ルートが「実質的に存在しない」状態
```

### 1-2. なぜ CPRM は「コピーから復号できない」のか

CPRM の鍵階層は、ユーザーが SDカードのファイルをコピーするだけでは鍵材料が手に入らない設計になっている。

```
デバイス鍵 (各機器に固有・非公開)
    + MKB (SDカードのシステム領域、AKE なしでは読めない)
    → メディア鍵 Km

Km + Media ID (SDカードの製造時固有値、AKE なしでは読めない)
    → メディア固有鍵 Kmu

Kmu + 暗号化タイトル鍵 (SDカード保護領域、AKE なしでは読めない)
    → タイトル鍵 (56bit C2 鍵)

タイトル鍵 + C2-CBC → 復号されたコンテンツ
```

- **MKB/MediaID/ETK はいずれも `dd`/通常コピーでは取得不可**
- カードを別のカードにコピーしても Media ID が変わるため復号不可
- デバイス鍵はメーカーがライセンスを受けて搭載する秘密値であり、公開されていない
- 実機（SH-06F）が壊れれば、正規手段での復号は完全に詰む

### 1-3. ワンセグが特に深刻な理由

フルHD放送（地上波・BS）の録画は、B-CAS + DLNA ムーブ等で一定の移行経路が存在する。
しかしワンセグ（SD-Video 規格）は:

- **対応機器が 2010 年前後の機種に限定**（Panasonic ゴリラ旧世代、SD-MobileImpact 等）
- **新しいカーナビやレコーダーは著作権保護 SD 動画に非対応**（「素の MP4 のみ」対応）
- **SD-Video 規格を再生できるソフトウェアが絶滅に近い**
- **HDMI キャプチャで実機から録画する」以外の救出手段がほぼない**

ワンセグは「低品質で価値が低い」と思われているかもしれないが、
**ドキュメンタリー・スポーツ中継・ニュース等の個人記録として利用されており、
その失効は利用者にとって非常に深刻**である。

## 2. 問題の構造的原因

### 2-1. 「正規手段の永続性」が保証されていない

DRM 設計において、**コンテンツ保護の観点からは成功**しているが、
**利用者のアクセス権保護の観点からは失敗**している。

| 設計要素 | 保護観点 | 利用者観点 |
|---------|---------|-----------|
| デバイス紐付け | ✅ 不正コピー防止 | ❌ 機器廃棄でアクセス消滅 |
| カード保護領域依存 | ✅ ファイルコピー防止 | ❌ カード破損で詰み |
| 配布ソフト依存 | ✅ 制御された復号 | ❌ 配布終了で詰み |
| OS 依存 | ✅ 制御された環境 | ❌ OS EOL で詰み |

### 2-2. 終了時の移行手段が設計されていない

CPRM/C2 を採用したシステムは、**「製品終了後にどう移行するか」** が設計に含まれていなかった。
DVD-VR 向け CPRM は鍵漏洩により復号ツールが普及したが、**SDカード向け CPRM は別鍵系統**であり、
同様の「事実上の解放」は発生していない。

### 2-3. 著作権法の硬直性

日本の著作権法は「技術的保護手段の回避」に対して厳格であるが、
**「保護手段が機能しなくなった場合の正規ユーザーのアクセス権」** に対する条項が不十分である。
アメリカの DMCA 1201(a) も同様の問題を持つが、米国では定期的な免除規定の見直し (Rulemaking) が行われている。

## 3. 既存規格 (ISDB-T ワンセグ) への対処

2026年時点でできる対処は限られている。以下を文書化・共有することを推奨する。

### 3-1. XP VM + SD-MobileImpact パッチによる救出

現存する唯一の実用的な手段。詳細は [`ワンセグ救出_再現手順.md`](ワンセグ救出_再現手順.md)。

- **Internet Archive に SD-MobileImpact の ISO が現存している**（2026年6月時点）
- 入手した状態でパッチを当てて XP VM で動かせば、**元 SD カードがあれば再生可能**
- **今すぐ実施すること**：将来的に ISO が削除されたり、VM ソフトが XP に対応しなくなる可能性がある

### 3-2. コミュニティへの呼びかけ

- XP VM 環境での SD-MobileImpact 動作確認情報の共有
- 他の SD-Video 対応機器（旧世代カーナビ等）の型番情報の収集
- 「元 SD カードを持っている限り」の救出手順の文書化

## 4. 将来規格 (ISDB-T3/ISDB-S3/地上波 4K) への提言

### 4-1. 提言の前提

次の世代の日本のデジタル放送録画 DRM として、以下の規格が展開予定・進行中:

| 規格 | 対象放送 | 状況 |
|------|---------|------|
| ISDB-T3 | 地上波 4K（将来） | 標準化議論中 |
| ISDB-S3 | BS 4K/8K | 2018年 12月放送開始済み |
| ARIBコンテンツ保護 | 地上波・BS の録画全般 | 現行継続 |

これらの録画保護設計において、**ワンセグ CPRM の失敗を繰り返さない**ために以下を提言する。

### 4-2. 提言 A: 機器紐付きの廃止または緩和

**現状の問題**: CPRM タイトル鍵がカード保護領域に格納され、
録画機器（デバイス鍵を持つもの）でしか復号できない。機器廃棄・故障でアクセス権喪失。

**提言**:
- 録画コンテンツの利用者所有権を明確化する法的枠組みを整備
- 「移行時の再暗号化」：機器廃棄前に、新しい機器・クラウドへの正規移行手段を必須化
- 「10年間の移行猶予」：規格終了後 10 年間は正規移行ソフトの配布を法的に義務付け
- CAS カードに相当するユーザー認証基盤（Personal Key）への移行検討

### 4-3. 提言 B: DRM ソフトウェアの長期サポート義務化

**現状の問題**: SD-MobileImpact は製品終了後わずか数年で動作不可能になり、
サポートソフトの配布も 2018 年に停止した。

**提言**:
- DRM ライセンス取得事業者に対し、**最終出荷から 10 年間の復号ソフトウェア維持を義務付ける**
- 維持が困難な場合の「鍵エスクロー」：規制機関（総務省・METI）への暗号鍵・手順の寄託
- 「インターオペラビリティ期間」の法制化：旧世代コンテンツへの新世代機器からのアクセス保証

### 4-4. 提言 C: 個人録画への自己復号権の明確化

**現状の問題**: 著作権法第30条（私的複製）は認められているが、
DRM による技術的制限の回避は「技術的保護手段の回避」として規制される。
この矛盾が、自己録画コンテンツへのアクセスすら困難にしている。

**提言**:
- 著作権法に「自己録画コンテンツの保存目的の技術的制限回避」の明示的な免除規定を追加
  （著作権法第30条の4 に相当する個人保存条項）
- アメリカ DMCA 1201(a) の 2-year Rulemaking 相当の定期免除規定を日本にも導入
- 「復号手段が事実上存在しない場合」の例外規定の明確化

### 4-5. 提言 D: オープンなアーカイブ形式への移行促進

**現状の問題**: ワンセグ (SD-Video/MPEG-4 AVC) もフルセグ (SHARP 独自) も、
復号後でなければ標準プレイヤーで再生できない独自暗号形式になっている。

**提言**:
- 録画ファイルの標準形式として MPEG-TS + 暗号オプション (標準コンテナ) を採用し、
  復号後の形式が特定ソフトに依存しない設計にする
- 10年後・20年後でも FFmpeg 等の汎用ソフトで再生可能な形式を目指す
- DRM を「コンテナではなく鍵管理層」に分離し、コンテンツ自体は標準形式で扱えるようにする

### 4-6. 提言 E: 緊急アーカイブ条項の法制化

**現状の問題**: 図書館・博物館による絶版作品のデジタル保存は著作権法で認められているが、
**放送録画の「絶版」相当状態（サービス終了・機器廃棄）への対応がない**。

**提言**:
- 「正規再生手段が事実上存在しない録画コンテンツ」については、
  公的機関（国立国会図書館・放送アーカイブ等）による保存目的の復号を認める
- 個人がその申請ルートを通じて、自己録画コンテンツの救出支援を受けられる制度の設計
- ISDB の国際標準としての地位を活かし、IEC/ISO での「録画コンテンツ保存指針」策定を推進

## 5. 当事者・関係機関への呼びかけ

### 録画機器メーカー（SHARP, Panasonic 等）

- 旧製品の CPRM 対応ソフトのソースコード、または鍵導出手順の公的アーカイブへの寄託を検討
- 「10年後の利用者」を想定した製品終了ポリシーの策定

### SD アソシエーション / 4C Entity

- CPRM 仕様の歴史的アーカイブ公開（研究目的での利用許諾）
- 旧世代 DRM の「日没条項」（sunset clause）の設計：N 年後に復号手順を公開する合意

### 総務省・METI

- 著作権法における「DRM 孤立化コンテンツ」の保護免除規定の立法化
- ISDB 次世代規格策定において、利用者のアクセス権保護を要件として明示
- 放送アーカイブ（NHK アーカイブス等）の対象に「個人録画の保存支援」の枠組みを検討

### ARIB（電波産業会）

- 次世代 ISDB-T3/S3 録画 DRM 設計に「サービス終了後 N 年での鍵公開」条項を標準化
- SD-Video 後継規格の移行経路の設計

---

## 6. まとめ

ワンセグ CPRM の問題は、DRM が「コンテンツを守ること」に成功しながら
「利用者の正当なアクセスを守ること」に失敗した典型例です。

2026 年に SH-06F で録画したワンセグ映像を見ようとすると:
1. Windows XP の VM を用意し
2. 絶版ソフトの ISO を Internet Archive で探し  
3. バイナリパッチを当てて
4. USB パススルーをセットアップして
5. OBS でリアルタイムにキャプチャする

という複雑な工程が必要です。これは **個人録画のデジタル権利が事実上消滅している**ことを示しています。

地上波 4K・BS 4K8K が普及する現在、今後の日本のデジタル放送録画規格には、
**20〜30年後の利用者が自分の録画データに普通にアクセスできること** を設計要件として組み込むことを強く求めます。
